231  /*
232  * @brief The releaseObject operation terminates execution of the application,
233  * returns all allocated computing resources, and de-allocates the
234  * resources’ capacities in use by the devices associated with the
235  * application. Before terminating, the application removes the message
236  * connectivity with its associated applications (e.g., ports, resources,
237  * and logs) in the domain.
238  *
239  * The following behavior is in addition to the LifeCycle::releaseObject operation
240  * behavior. The Application::releaseObject operation shall release each
241  * application component not created by a resource factory by utilizing the
242  * component’s Resource::releaseObject operation. The Application::releaseObject
243  * operation shall release each component created by a resource factory via the
244  * ResourceFactory::releaseResource operation. The Application::releaseObject
245  * operation shall terminate a resource factory when no more resources are managed
246  * by the resource factory via the ResourceFactory::shutdown operation. The
247  * Application::releaseObject operation shall terminate the processes / tasks on
248  * allocated executable devices belonging to each application component by
249  * utilizing the ExecutableDevice:terminate operation. The releaseObject operation
250  * shall de-allocate the memory associated with each application component
251  * instance from its allocated device by utilizing the LoadableDevice::unload
252  * operation.
253  *
254  * The releaseObject operation shall deallocate the device capacities that were
255  * allocated during application creation. The actual devices deallocated
256  * (Device::deallocateCapacity) reflect changes in their capacity based upon
257  * component capacity requirements deallocated from them, which may also cause
258  * state changes for the devices.
259  *
260  * The application shall release all object references to the components making
261  * up the application.
262  *
263  * The releaseObject operation shall disconnect ports that were previously
264  * connected based upon the application’s software profile.
265  *
266  * The releaseObject operation shall disconnect consumers and producers from a
267  * CORBA Event Service’s event channel based upon the software profile. The
268  * releaseObject operation may destroy a CORBA Event Service’s event channel when
269  * no more consumers and producers are connected to it.
270  *
271  * For components (e.g., Resource, ResourceFactory) that are registered with
272  * Naming Service, the releaseObject operation shall unbind those components and
273  * destroy the associated naming contexts as necessary from the Naming Service.
274  *
275  * The releaseObject operation for an application shall disconnect ports first,
276  * then release the resources and the resource factories, then call the terminate
277  * operation, and lastly call the unload operation on the devices.
278  *
279  * The releaseObject operation shall, upon successful application release,
280  * write an ADMINISTRATIVE_EVENT log record.
281  *
282  * The releaseObject operation shall, upon unsuccessful application release,
283  * write a FAILURE_ALARM log record.
284  *
285  * The releaseObject operation shall send a DomainManagementObjectRemovedEventType
286  * event to the Outgoing Domain Management event channel upon successful release
287  * of an application. For this event,
288  * 1. The producerId is the identifier attribute of the released application.
289  * 2. The sourceId is the identifier attribute of the released application.
290  * 3. The sourceName is the name attribute of the released application.
291  * 4. The sourceCategory is “APPLICATION”.
292  *
293  * The following steps demonstrate one scenario of the application’s behavior
294  * for the release of an application that contains ResourceFactory behavior:
295  * 1. Client invokes releaseObject operation.
296  * 2. Disconnect Ports.
297  * 3. Release the ResourceFactory components.
298  * 4. Shutdown the ResourceFactory components.
299  * 5. Release the Resource components.
300  * 6. Terminate the components’ processes.
301  * 7. Unload the components’ executable images.
302  * 8. Change the state of the associated devices to be available, along with
303  * device(s) memory utilization availability and processor utilization
304  * availability based upon the Device Profile and software profile.
305  * 9. Unbind application components from Naming Service.
306  * 10. Log an Event indicating that the application was either successfully
307  * or unsuccessfully released.
308  * 11. Remove the application reference from the applications attribute
309  * and generate an event to indicate the application has been removed
310  * from the domain.
311  *
312  * @exception The releaseObject operation shall raise a ReleaseError exception
313  * when internal processing errors prevent the successful release of
314  * any application component.
315  */
